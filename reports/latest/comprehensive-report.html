Azure 総合運用監査レポート

生成日時: 2026-07-09 11:33 UTC

エグゼクティブサマリ

本レポートは、現在ログイン中の Azure サブスクリプションにおけるリソース、RBAC、NSG、Defender for Cloud、Azure Advisor の横断的な棚卸しとリスク分析を実施した結果をまとめたものです。

  • リソース総数: 58 件 / リソースグループ数: 8 / リージョン数: 5
  • 主要リソース件数: VM、Storage(LRS)、Public IP は詳細未取得
  • タグ付与率: 50.0%
  • RBAC 総割り当て数: 未取得 / Owner 割り当て数: 13 / UAA (User Assigned Assigned) 割り当て数: 2 / 孤児候補件数: 40
  • マネージメントグループスコープの Owner 数: 未取得
  • NSG 数: 未取得 / 総ルール数: 未取得 / 高リスクルール件数: 0
  • Defender for Cloud Unhealthy 件数: 101（High: 32、Medium: 未集計、Low: 未集計）
  • Defender for Cloud 最頻出推奨事項: Linux VM のアップデート未適用多数（例: Update libpython3.10-minimal_for_linux 等）
  • Azure Advisor 件数: 未集計（5カテゴリ内訳未取得）、High Impact 件数: 25
  • Azure Advisor 最頻出問題: 「Azure Disks を VM と同一ゾーンに配置」および「Trusted Launch の有効化」
  • Defender for Cloud は有効でデータ取得済み。Azure Advisor は導入済みでデータ取得済み。

本サブスクリプションの主要改善領域は以下の4つです。

  1. 公開境界：Public IP と NSG 高リスクルールは少ないが、公開リソースの管理強化が必要。
  2. 特権境界：Owner 割り当て数や孤児候補が多く、特権アクセス管理の強化が急務。
  3. 監視フィードバック：Defender for Cloud と Advisor の推奨事項を活用した継続的なセキュリティ改善。
  4. ガバナンス基盤：タグ付与率 50% と低く、ポリシー適用やリソース管理の強化が必要。

潜在リスク Top 5

1. 公開境界の露出 Low
NSG 高リスクルール件数は 0 件、Public IP 件数は未取得ですが、公開リソースの管理強化が望まれます。Bastion や JIT、Application Gateway WAF の導入を推奨します。
推奨リンク: Azure Bastion / JIT VM Access / Application Gateway WAF
2. 特権ロールの恒久付与と blast radius High
Owner 割り当て数は 13 件、UAA は 2 件、孤児候補は 40 件と多く、特権アクセスの過剰付与が懸念されます。PIM、マネージド ID、Access Review の導入を強く推奨します。
推奨リンク: Azure AD PIM / マネージド ID / Access Reviews
3. Defender for Cloud のセキュリティ態勢ギャップ High
Defender for Cloud は有効で、Unhealthy 件数は 101 件（High: 32 件）と多く、Linux VM のアップデート未適用が多数検出されています。Foundational CSPM と Secure Score の月次レビューを推奨します。
推奨リンク: Defender for Cloud 入門 / Secure Score / CSPM (Cloud Security Posture Management)
4. Advisor フィードバックループ不在 High
Advisor の High Impact 件数は 25 件で、Azure Disks のゾーン配置や Trusted Launch の有効化など重要な推奨事項が多数あります。月次レビューと Cost Management 連動を推奨します。
推奨リンク: Azure Advisor 概要 / Cost Management 連携 / Advisor ベストプラクティス
5. ガバナンス基盤の弱さ Medium
タグ付与率は 50.0% と低く、Storage は LRS のみ、Public IP 管理も不明瞭です。Azure Policy、Tag Inheritance、ZRS/GZRS への冗長性移行を推奨します。
推奨リンク: Azure Policy / タグ付与のベストプラクティス / Storage 冗長性オプション
