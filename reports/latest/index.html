主要改善領域

  1. 公開境界：NSG 高リスクルールは0件ですが、Public IP 件数未取得のため潜在的な公開リスクは未評価。Bastion + JIT、Application Gateway WAF の導入推奨。
  2. 特権境界：Owner 13件、孤児候補 40件と特権ロールの恒久付与および孤児アカウントの存在が懸念。PIM、マネージドID、Access Review の実施推奨。
  3. 監視フィードバック：Defender for Cloud で High 31件の不健康状態が存在。Foundational CSPM と Secure Score の月次レビューが必要。
  4. ガバナンス基盤：タグ付与率 45.1% と低く、ストレージ冗長性や Public IP 管理も不十分。Azure Policy、Tag Inheritance、ZRS/GZRS への移行推奨。

1. 公開境界の露出 Low

2. 特権ロールの恒久付与と blast radius High
Owner 割り当て数は 13件、孤児候補は 40件 と多く、特権ロールの恒久付与や不要なアクセス権の放置が懸念されます。
推奨: PIM (Privileged Identity Management)、マネージドIDの活用、定期的な Access Review の実施を強く推奨します。
Azure AD PIM マネージドID Access Reviews
3. Defender for Cloud のセキュリティ態勢ギャップ High
Defender for Cloud の Unhealthy 件数は 79件、うち High Severity は 31件 あります。セキュリティ態勢に重要なギャップが存在します。
推奨: Foundational CSPM の導入と Secure Score の月次レビューを実施し、継続的な改善を図ってください。
Defender for Cloud 入門 Secure Score CSPM 概要
4. Advisor フィードバックループ不在 High
Advisor の High Impact 件数は 25件 あり、ストレージ冗長化や可用性向上に関する推奨が多く見られます。Advisor の推奨を活用した運用改善が不足しています。
