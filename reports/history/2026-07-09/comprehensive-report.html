生成日時: 2026-07-09 20:32:01 UTC

エグゼクティブサマリ

本レポートは、現在ログイン中の Azure サブスクリプションにおけるリソース、RBAC、NSG、Defender for Cloud、Azure Advisor の横断的な棚卸しとリスク分析を実施した結果をまとめたものです。

主要数値サマリ

  • リソース総数: 75 件 / リソースグループ数: 9 / リージョン数: 5
  • VM件数: データなし / Storage (LRS) 件数: データなし / Public IP 件数: データなし
  • タグ付与率: 52.0%
  • RBAC 総割り当て: データなし / Owner 割り当て: 13 / UAA (User Assigned Assigned) 割り当て: 2 / 孤児候補件数: 40 / Management Group スコープの Owner: データなし
  • NSG 数: データなし / 総ルール数: データなし / 高リスクルール件数: 0
  • Defender for Cloud Unhealthy 件数: 178（High: 53 / Medium: 未集計だが多数 / Low: 未集計だが少数）
  • Defender 最頻出推奨事項: Update linux-image-generic_for_linux 等パッチ適用関連
  • Azure Advisor 件数: 30（High Impact 件数: 30）
  • Advisor 最頻出問題: Linux 仮想マシンの Azure Disk Encryption または EncryptionAtHost の有効化推奨
  • Defender for Cloud は有効化済み、Azure Advisor は導入済みのためデータ取得は正常に行われています。

主要改善領域

  1. 公開境界の強化 - Public IP と NSG 高リスクルールの監視と制御
  2. 特権境界の管理 - Owner、UAA、孤児アカウントの整理と PIM 適用
  3. 監視フィードバックの活用 - Defender と Advisor の推奨事項を活かした運用改善
  4. ガバナンス基盤の強化 - タグ付与率向上とストレージ冗長性の最適化

潜在リスク Top 5

1. 公開境界の露出 Low

本サブスクリプションでは NSG の高リスクルールは 0件、Public IP 件数は未取得のため詳細不明ですが、公開境界の露出は現状低リスクと評価されます。
推奨: Bastion と JIT (Just-In-Time) アクセスの導入、Application Gateway WAF の活用を検討してください。

参考リンク: Azure Bastion / JIT VM Access / Application Gateway WAF

2. 特権ロールの恒久付与と blast radius Medium

Owner 割り当てが 13件、UAA 割り当てが 2件、孤児候補が 40件存在し、特権ロールの恒久付与によるリスクが懸念されます。
推奨: PIM (Privileged Identity Management) の適用、マネージド ID の活用、定期的な Access Review を実施してください。

参考リンク: Azure AD PIM / マネージド ID / Access Reviews

3. Defender for Cloud のセキュリティ態勢ギャップ High

Defender for Cloud の Unhealthy 件数は 178件、うち High Severity が 53件存在し、セキュリティ態勢に重大なギャップがあります。
推奨: Foundational CSPM (Cloud Security Posture Management) の導入、Secure Score の月次レビューを必須化してください。

参考リンク: Defender for Cloud 概要 / Secure Score / CSPM のベストプラクティス

4. Advisor フィードバックループ不在 High

Azure Advisor の High Impact 推奨が 30件存在し、フィードバックループが不十分な可能性があります。
推奨: Advisor 推奨事項の月次レビューを実施し、Cost Management と連動した改善活動を推進してください。

参考リンク: Azure Advisor 概要 / Cost Management 連携 / Advisor ベストプラクティス

5. ガバナンス基盤の弱さ Medium

タグ付与率は 52.0% と半数程度に留まり、ストレージの冗長性や Public IP 管理も不十分な可能性があります。
推奨: Azure Policy によるタグ必須化、Tag Inheritance の導入、LRS から ZRS/GZRS への冗長性移行を推進してください。

参考リンク: Azure Policy / タグ管理 /
